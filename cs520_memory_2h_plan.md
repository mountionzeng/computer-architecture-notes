## 📅 第2小时: 分页机制详解 (14:30-15:30)

### 🎯 学习目标
- 掌握分页的核心原理
- 理解页表结构与地址转换

### 📖 内容与时间分配

**14:30-14:50 | Paging 基础 ⚠️核心**
- [ ] **Page (页)**: 逻辑内存的固定大小块 (通常4KB)
- [ ] **Frame (帧)**: 物理内存的固定大小块
- [ ] **Page Table (页表)**: 页到帧的映射表
- [ ] **地址转换过程**:
  ```
  逻辑地址 = 页号(p) + 页内偏移(d)
  物理地址 = 帧号(f) + 页内偏移(d)
  ```
- [ ] **计算示例**:
  - 页大小 = 4KB = 4096 bytes
  - 逻辑地址 5000:
    - 页号 p = 5000 / 4096 = 1
    - 偏移 d = 5000 % 4096 = 904
    - 如果页表[1] = 帧6，则物理地址 = 6×4096 + 904 = 25480
- **关键点**: 分页消除外部碎片，但有内部碎片

**14:50-15:10 | TLB (Translation Lookaside Buffer)**
- [ ] **TLB原理**: 页表的快速缓存
- [ ] **TLB工作流程**:
  1. 检查TLB中是否有页号
  2. TLB命中 → 直接获取帧号
  3. TLB未命中 → 查页表，更新TLB
- [ ] **有效访问时间 (EAT) 计算**:
  ```
  EAT = (命中率 × TLB时间) + (未命中率 × (TLB时间 + 页表时间 + 内存时间))
  ```
- [ ] **示例**: 
  - TLB命中率80%，TLB访问20ns，内存访问100ns
  - EAT = 0.8×20 + 0.2×(20+100+100) = 16 + 44 = 60ns
- **关键点**: TLB是硬件优化，显著减少地址转换开销

**15:10-15:30 | Page Table Structure + 总结**
- [ ] **页表项结构**:
  - 帧号 (Frame number)
  - Valid-Invalid bit (有效位)
  - Protection bits (保护位: read/write/execute)
  - Reference bit (引用位)
  - Dirty bit (修改位)
- [ ] **Valid-Invalid bit**:
  - Valid (1): 页在内存中
  - Invalid (0): 页不在进程地址空间或被换出
- [ ] **分页优缺点总结**:
  - ✅ 优势: 消除外部碎片，允许非连续物理内存
  - ⚠️ 代价: 内部碎片(平均半页)，页表占用内存
- [ ] **手算练习**: 
  - 给定页大小和逻辑地址，计算物理地址
  - 给定TLB命中率，计算EAT

## 📝 重点公式

### 分页地址转换
```
页号 p = 逻辑地址 / 页大小
偏移 d = 逻辑地址 % 页大小
物理地址 = PageTable[p] × 页大小 + d
```

### TLB有效访问时间
```
EAT = h × t_TLB + (1-h) × (t_TLB + t_PT + t_mem)
h = TLB命中率
```
