# 计算机架构笔记

> 系统性学习现代软件架构的完整知识体系

## 📚 文档目录

### [软件架构完整体系](./architecture-system.md)

从宏观到微观，系统梳理软件架构的完整知识图谱：

- **宏观分层架构**：单体、分层、微服务、服务网格、Serverless
- **前端架构**：MPA/SPA、SSR/SSG/ISR、微前端
- **后端架构模式**：MVC、DDD、六边形架构、CQRS、事件溯源
- **数据架构**：数据库选型、缓存架构、数据仓库与数据湖
- **分布式架构**：CAP 理论、消息队列、分布式事务
- **云原生架构**：容器化、Kubernetes、CI/CD、可观测性

### [计算机架构核心概念详解](./architecture-concepts.md)

深入解析软件系统中的9个核心概念：

1. **页面（Page）**：业务视图的顶层容器
2. **组件（Component）**：UI 的最小可复用单元
3. **接口（API / Interface）**：系统间通信的契约
4. **表结构（Database Schema）**：数据持久化的设计蓝图
5. **环境变量（Environment Variables）**：配置与代码分离
6. **数据流（Data Flow）**：数据在系统中的流转路径
7. **路由（Routing）**：URL 与逻辑的映射关系
8. **权限（Authorization & Authentication）**：认证身份 + 授权操作
9. **部署方式（Deployment）**：代码到服务的交付过程

## 🎯 学习路径建议

2. 再学习《架构完整体系》中的"单体架构"和"分层架构"

1. 深入学习《架构完整体系》中的"微服务架构"和"分布式架构"
2. 实践"后端架构模式"中的 DDD 和 CQRS
3. 搭建完整的 CI/CD 流水线

### 架构师
1. 系统学习完整的架构体系
2. 重点关注"架构选型决策框架"
3. 根据业务场景设计合适的架构演进路线

## 📖 架构演进路线图

```
阶段一：单体起步
代码 → 单一进程 → 单一数据库 → 直接部署服务器

阶段二：分层优化
表现层 / 业务层 / 数据层分离 → 引入缓存 → CDN 加速

阶段三：服务化拆分
API Gateway → 核心服务独立 → 消息队列解耦 → 读写分离

阶段四：云原生成熟
容器化 → K8s 编排 → CI/CD → 服务网格 → 完整可观测性

阶段五：平台化演进
多云部署 → 混沌工程 → AIOps → 自愈系统
```

## 💡 核心原则

- **不要过度设计**：用最简单的架构满足当前需求
- **康威定律**：系统架构往往反映团队的沟通结构
- **渐进式迁移**：从单体到微服务是逐步演进，不是一次性重写
- **可观测性优先**：在系统复杂化之前建立监控体系
- **拥抱失败**：分布式系统中故障是常态
