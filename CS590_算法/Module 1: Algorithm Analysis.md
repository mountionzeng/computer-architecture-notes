## 1. 思考脉络（为什么需要这套工具）

1. **为什么不能用秒数比较算法？** —— 机器、语言、输入规模都不一样，秒数没有可比性
2. → 抽象出**基本操作数** $T(n)$ 作为代价函数（对内存/CPU 中立）
3. → $T(n)$ 太具体，关心**增长趋势** → 渐进记号 $O / \Omega / \Theta$
4. → 同一个算法在不同输入下表现不同 → **最坏 / 平均 / 最好情况**
5. → 怎么从代码里读出复杂度？→ **循环计数规则**
6. → 递归算法怎么办？→ **递推式求解**（代入法 / 递归树 / 主定理）
7. → 单次操作很慢，但平均起来很快？→ **摊还分析**（聚合 / 记账 / 势能）

## 2. 渐进记号（Asymptotic Notation）

### 2.1 五大记号速查

| 记号 | 中文 | 直觉 | 形式定义 | 用法 |
|---|---|---|---|---|
| $O(g(n))$ | 大 O | **上界**（至多这么慢） | $\exists c, n_0:\ 0 \leq f(n) \leq c \cdot g(n),\ \forall n \geq n_0$ | 最常用，描述最坏情况 |
| $\Omega(g(n))$ | 大 Omega | **下界**（至少这么慢） | $\exists c, n_0:\ 0 \leq c \cdot g(n) \leq f(n),\ \forall n \geq n_0$ | 证明"不可能更快" |
| $\Theta(g(n))$ | 大 Theta | **同阶** | $f$ 既是 $O(g)$ 又是 $\Omega(g)$ | $O$ 和 $\Omega$ 同时成立 |
| $o(g(n))$ | 小 o | **严格慢于** $g$ | $\lim_{n \to \infty} f/g = 0$ | 比 $O$ 更严格（严格上界） |
| $\omega(g(n))$ | 小 omega | **严格快于** $g$ | $\lim_{n \to \infty} f/g = \infty$ | 比 $\Omega$ 更严格（严格下界） |

### 2.2 三个常被忽略的性质

- **传递性**：$f = O(g)$ 且 $g = O(h)$ → $f = O(h)$
- **加法吃大**：$O(f) + O(g) = O(\max(f, g))$（如 $O(n) + O(n^2) = O(n^2)$）
- **乘法分配**：$O(f) \cdot O(g) = O(f \cdot g)$（嵌套循环就是这么来的）

## 3. 最坏 / 平均 / 最好情况

同一个算法在不同输入下表现不一样，必须明确分析的是哪种：

| 类型 | 定义 | 何时关心 | 例子（线性查找） |
|---|---|---|---|
| **最坏情况**（worst case） | 所有输入中跑得最慢的那种 | 工程默认；安全性敏感场景 | 目标在末尾或不存在 → $\Theta(n)$ |
| **平均情况**（average case） | 所有输入按概率分布的期望 | 算法竞赛、统计性能 | 假设均匀分布 → $\Theta(n/2) = \Theta(n)$ |
| **最好情况**（best case） | 跑得最快的那种 | 几乎不用，容易误导 | 目标在第一位 → $\Theta(1)$ |

> **关键认知**：摊还分析的"平均"和这里的"平均"**不是一回事**。摊还是**最坏情况下的多次操作平均**，不依赖输入概率分布。

## 4. 常见复杂度阶（Complexity Classes）

```
O(1) < O(log n) < O(√n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
```

**注意 y 轴是对数刻度**——这点很关键。如果用普通刻度，$2^n$ 和 $n!$ 会冲到天上去，把其他所有线全压成贴在 x 轴上的一条平线，根本看不出区别。即便用了对数刻度，$n!$ 仍然甩开其他函数一大截。

### 4.1 四档直觉

| 档位 | 复杂度 | 实战表现 | 典型算法 |
|---|---|---|---|
| 🟢 **第一档（瞬时）** | $O(1)$、$O(\log n)$、$O(\sqrt{n})$ | $n$ 即使十亿也几乎不耗时 | 哈希查找、二分查找 |
| 🟡 **第二档（线性）** | $O(n)$、$O(n \log n)$ | GB 级数据没问题 | 归并 / 快排 / 堆排 |
| 🟠 **第三档（小数据可用）** | $O(n^2)$、$O(n^3)$ | $n$ 上千还行，上百万就崩 | 冒泡 / 插排 / 朴素矩阵乘法 |
| 🔴 **第四档（指数级）** | $O(2^n)$、$O(n!)$ | 只能处理极小 $n$（$n \approx 30$ 就要小时级） | NP-hard 类问题（TSP 暴力解） |

|---|---|

| $n \leq 20$ | $O(2^n)$ |
| $n \leq 500$ | $O(n^3)$ |
| $n \leq 5000$ | $O(n^2)$ |
| $n \leq 10^6$ | $O(n \log n)$ |
| $n \leq 10^8$ | $O(n)$ |
| $n > 10^8$ | $O(\log n)$、$O(1)$ |

## 5. 怎么算一段代码的复杂度

### 5.1 五条计数规则

1. **顺序结构**：相加，取最大 → $O(f) + O(g) = O(\max(f, g))$
2. **循环**：循环次数 × 循环体复杂度
3. **嵌套循环**：相乘
4. **条件分支**：取最坏分支
5. **递归调用**：写出递推式后求解（见第 6 节）

### 5.2 速读模板

```python
# 例 1: O(n) — 单层循环
for i in range(n):
    ...                    # O(1)

# 例 2: O(n²) — 嵌套循环
for i in range(n):
    for j in range(n):
        ...                # O(1)

# 例 3: O(n log n) — 外循环 n 次，内循环 log n
for i in range(n):
    j = 1
    while j < n:
        j *= 2             # 每次翻倍 → log n 次

# 例 4: O(log n) — 折半
while n > 1:
    n //= 2

# 例 5: 易错！O(n) 不是 O(n²)
# 外循环跑 n 次，内循环总共也只跑 n 次（i+j 累计 ≤ 2n）
i, j = 0, 0
while i < n:
    while j < n and ...:
        j += 1
    i += 1
```

### 5.3 易错点：循环变量增长方式决定复杂度

| 增长方式 | 循环次数 |
|---|---|
| `i += 1` | $n$ |
| `i += k`（k 是常数） | $n/k = \Theta(n)$ |
| `i *= 2` 或 `i /= 2` | $\log_2 n$ |
| `i *= k`（k > 1） | $\log_k n = \Theta(\log n)$ |
| `i = i*i` | $\log \log n$（增长极慢） |

## 6. 递推式求解（Recurrence Solving）

递归算法的复杂度写成递推式 $T(n) = a \cdot T(n/b) + f(n)$（或其他形式）。三种解法：

### 6.1 代入法（Substitution / 数学归纳）

**步骤**：猜一个解 → 用归纳法证明。

**例**：$T(n) = 2T(n/2) + n$

猜 $T(n) = O(n \log n)$，即 $T(n) \leq c \cdot n \log n$。
代入：$T(n) \leq 2c(n/2)\log(n/2) + n = cn\log n - cn + n \leq cn \log n$（当 $c \geq 1$）✓

**适用**：已经有猜测时验证；初学者不友好。

### 6.2 递归树法（Recursion Tree）

**步骤**：把递归展开成树，统计每层代价 + 树高。

**例**：$T(n) = 2T(n/2) + n$
- 第 0 层：$n$
- 第 1 层：$2 \times (n/2) = n$
- 第 $k$ 层：$2^k \times (n/2^k) = n$
- 总层数：$\log_2 n$
- **总代价**：$n \times \log n = \Theta(n \log n)$ ✓

**适用**：直观、能猜出答案。常作为代入法的"猜测来源"。

### 6.3 主定理（Master Theorem）⭐

对 $T(n) = a \cdot T(n/b) + f(n)$，其中 $a \geq 1$、$b > 1$，比较 $f(n)$ 与 $n^{\log_b a}$：

> **核心思想**：本层额外工作 $f(n)$ 与纯递归代价 $n^{\log_b a}$ 谁更"重"，谁就主导最终复杂度。
>
> **关键量** $n^{\log_b a}$ **的含义**：递归树的叶子总数。即"完全不做合并工作时"的纯递归开销。

#### Case 1：递归代价更重（叶子主导）

- **条件**：$f(n) = O(n^{\log_b a - \varepsilon})$，即 $f$ 比 $n^{\log_b a}$ **多项式级别地小**
- **结论**：$T(n) = \Theta(n^{\log_b a})$
- **直觉**：每层额外工作越来越少，叶子层主导
- **例**：$T(n) = 2T(n/2) + O(1)$（二叉树遍历）
  - $a=2, b=2$，$n^{\log_2 2} = n$
  - $f(n) = O(1)$，满足 Case 1
  - 结论：$T(n) = \Theta(n)$

#### Case 2：势均力敌（每层贡献相等）

- **条件**：$f(n) = \Theta(n^{\log_b a})$
- **结论**：$T(n) = \Theta(n^{\log_b a} \cdot \log n)$
- **直觉**：每层代价相同，共 $\log n$ 层
- **例**：$T(n) = 2T(n/2) + O(n)$（归并排序）
  - $a=2, b=2$，$n^{\log_2 2} = n$
  - $f(n) = \Theta(n)$，满足 Case 2
  - 结论：$T(n) = \Theta(n \log n)$ ✓

#### Case 3：额外工作更重（根主导）

- **条件**：$f(n) = \Omega(n^{\log_b a + \varepsilon})$，且**正则条件**：$a \cdot f(n/b) \leq c \cdot f(n)$（$c < 1$）
- **结论**：$T(n) = \Theta(f(n))$
- **直觉**：根层工作主导，子问题可忽略
- **例**：$T(n) = 2T(n/2) + O(n^2)$
  - $f(n) = n^2 = \Omega(n^{1+1})$，满足 Case 3
  - 正则条件：$2 \cdot (n/2)^2 = n^2/2 \leq \frac{1}{2} n^2$ ✓
  - 结论：$T(n) = \Theta(n^2)$

#### 三种 Case 对比

| Case | 谁更重？ | 结论 | 类比 |
|---|---|---|---|
| Case 1 | 叶子（递归）更重 | $\Theta(n^{\log_b a})$ | 员工比老板做得多 |
| Case 2 | 平分秋色 | $\Theta(n^{\log_b a} \cdot \log n)$ | 每层贡献相等 |
| Case 3 | 根（额外工作）更重 | $\Theta(f(n))$ | 老板比员工做得多 |

> ⚠️ **盲区**：如果 $f(n)$ 与 $n^{\log_b a}$ 差的不是多项式倍（比如只差 $\log$ 倍，例 $T(n) = 2T(n/2) + n \log n$），三个 Case 都不适用。这种情况要用递归树法。

### 6.4 三种方法选用

| 方法 | 何时用 | 难度 |
|---|---|---|
| 主定理 | 形式严格匹配 $aT(n/b) + f(n)$ | 简单（套公式） |
| 递归树 | 主定理盲区，或想直观理解 | 中等 |
| 代入法 | 已有猜测要验证 | 较难 |

## 7. 摊还分析（Amortized Analysis）

### 7.1 为什么需要

有些数据结构操作**偶尔很贵，但大多数时候很便宜**。最坏情况会高估真实代价。

**经典例子：动态数组（Python list / Java ArrayList）的 append**
- 通常：直接写入，$O(1)$
- 偶尔：扩容时复制全部元素，$O(n)$

最坏情况说"每次 append 是 $O(n)$"——**严重高估**。摊还分析回答："$n$ 次操作平均每次的真实代价是多少？"

> ⚠️ **摊还 ≠ 平均情况分析**。摊还是**最坏输入下的多次操作平均**，不依赖输入概率分布。

### 7.2 方法一：聚合法（Aggregate Method）

**思路**：直接算 $n$ 次操作的总代价 $T(n)$，再除以 $n$。

$$\text{摊还代价} = \frac{T(n)}{n}$$

**动态数组扩容分析**：
- 扩容发生在第 1, 2, 4, 8, ..., $n$ 次（每次容量翻倍）
- 复制代价：$1 + 2 + 4 + \cdots + n = 2n - 1 = O(n)$
- 加上 $n$ 次普通写入 $O(n)$
- 总 $T(n) = O(n)$ → 摊还 = $O(n)/n = O(1)$ ✓

**优点**：简单直接
**缺点**：所有操作给同一个摊还代价，不适合多种操作

### 7.3 方法二：记账法（Accounting Method）

**思路**：给每种操作"收费"（可以高于实际代价），多收的存入"银行"，未来贵的操作从银行取钱补贴。

**规则**：
- 银行余额 $\geq 0$（不能透支）
- 总实际代价 $\leq$ 总收费

**动态数组分析**：
- 每次 append 收 **3 元**：1 元写入 + 2 元存银行
- 扩容时复制 $n/2$ 个旧元素：每个之前存了 2 元，共 $n$ 元，刚好够
- 摊还代价 = 3 = $O(1)$

**优点**：直观，像预付费
**缺点**：需要灵感选收费方案

### 7.4 方法三：势能法（Potential Method）⭐ 最严谨

**思路**：定义势函数 $\Phi$（衡量数据结构当前"蓄势状态"）：

$$\hat{c}_i = c_i + \Delta\Phi = c_i + \Phi_i - \Phi_{i-1}$$

| 符号 | 含义 |
|---|---|
| $c_i$ | 第 $i$ 次操作的**实际代价** |
| $\hat{c}_i$ | 第 $i$ 次操作的**摊还代价** |
| $\Phi_i$ | 第 $i$ 次操作后的**势能** |
| $\Delta\Phi$ | 势能变化量（可正可负） |

**为什么这样定义？** $n$ 次摊还代价求和：

$$\sum_{i=1}^n \hat{c}_i = \sum_{i=1}^n c_i + \Phi_n - \Phi_0$$

只要 $\Phi_n \geq \Phi_0$，就有 $\sum c_i \leq \sum \hat{c}_i$，**总摊还代价是总实际代价的上界**。

**动态数组分析**：定义 $\Phi = 2 \times \text{元素数} - \text{容量}$
- 普通 append（$k < m$）：$c=1$，$\Delta\Phi = +2$，摊还 = 3
- 扩容 append（$k = m$）：$c = m+1$，$\Delta\Phi = 2 - m$，摊还 = $(m+1) + (2-m) = 3$

两种情况都是 $O(1)$ ✓

**优点**：最严谨通用，适合复杂结构（红黑树、splay tree）
**缺点**：找好的势函数需要经验

### 7.5 三种方法对比

| 方法 | 思路 | 难度 | 适用场景 |
|---|---|---|---|
| **聚合法** | 总代价 ÷ 次数 | 最简单 | 操作单一 |
| **记账法** | 预收费存银行 | 中等 | 操作种类少 |
| **势能法** | 势函数量化"蓄力" | 最复杂 | 复杂结构 |

> 🔗 **本质相同**：记账法的"银行存款" ≡ 势能法的 $\Phi$。势能法是记账法的数学形式化版本。

## 8. 空间复杂度（简介）

类似时间复杂度，但衡量**额外内存**（不含输入本身）：

| 算法 | 时间 | 空间 |
|---|---|---|
| 二分查找（迭代） | $O(\log n)$ | $O(1)$ |
| 二分查找（递归） | $O(\log n)$ | $O(\log n)$ —— 递归栈 |
| 归并排序 | $O(n \log n)$ | $O(n)$ —— 临时数组 |
| 快速排序 | $O(n \log n)$ | $O(\log n)$ —— 递归栈 |
| 动态规划（典型） | $O(n^2)$ | $O(n^2)$ 或滚动数组优化到 $O(n)$ |

**时空权衡**：很多算法可以**用空间换时间**（哈希表加速查找）或**用时间换空间**（不存中间结果，重新计算）。

## 10. 常考陷阱

- ❌ $O(2n) = O(n)$，常数可吃；但**底数**不可吃（$2^n \neq 3^n$）
- ❌ $\log$ 的底数在 $O$ 记号下可忽略；但**指数**不可忽略
- ❌ 主定理 Case 3 必须验证正则条件
- ❌ 主定理只适用于 $T(n) = aT(n/b) + f(n)$ 形式，**不能套到** $T(n) = T(n-1) + f(n)$
- ❌ 摊还 ≠ 平均：摊还是最坏输入下的平均，不依赖输入分布
- ❌ "best case" 几乎没分析价值，工程上不要拿来宣传性能

## 11. 面试高频题

1. **解释 $O / \Omega / \Theta$ 区别** —— 上界/下界/同阶 + 形式定义
2. **$T(n) = 2T(n/2) + n$ 是多少？** —— Master Theorem Case 2，$\Theta(n \log n)$
3. **动态数组 push 的摊还复杂度？** —— $\Theta(1)$，记账法每次预付 3 单位
4. **$\log_2 n$ 和 $\log_{10} n$ 在 $O$ 记号下相同吗？** —— 相同，差常数倍
5. **主定理三种情况各举一个例子** —— 二叉树遍历 / 归并排序 / Strassen 矩阵乘法
6. **能用主定理解 $T(n) = T(n-1) + n$ 吗？** —— 不能，形式不符；用展开法 → $\Theta(n^2)$
7. **快排平均 $O(n \log n)$ 但最坏 $O(n^2)$，怎么避免？** —— 随机化主元 / 三数取中
8. **摊还分析三种方法等价吗？** —— 等价，记账法的银行 ≡ 势能法的 $\Phi$

## 14. 一句话总结

- **渐进记号**：把秒数转成"增长趋势"，$O$ 上界、$\Omega$ 下界、$\Theta$ 同阶
