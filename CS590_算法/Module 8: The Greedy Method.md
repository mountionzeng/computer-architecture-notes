## 1. 思考脉络

1. 暴力穷举太慢 → 能否每步只做一次"局部最优"决策？
2. 若**贪心选择 + 最优子结构**成立 → 贪心可达全局最优
3. 不成立 → 需 DP（Module 10）或回溯
4. 证明贪心正确性的经典手法：
   - **交换论证（Exchange Argument）**
   - **数学归纳**
   - **保持不变式**

---

## 2. 核心知识点

### 2.1 贪心算法成立的两个性质

1. **贪心选择性质**：局部最优选择能导向全局最优
2. **最优子结构**：全局最优解包含子问题的最优解

若两者都成立 → 贪心；若只有 (2) → DP。

### 2.2 Fractional Knapsack（分数背包）

- 容量 W，物品 (wᵢ, vᵢ)，可取任意比例
- **贪心**：按 vᵢ/wᵢ 降序取
- **O(n log n)**，贪心最优
- ⚠️ 0-1 背包**不能用贪心**（见 Module 10）

### 2.3 Activity Selection（活动选择）

- 给一组 (start, end) 区间，选最多**不重叠**的活动
- **贪心**：按 **end 升序**排序，依次选不冲突的
- O(n log n)

### 2.4 Task Scheduling（任务调度）

- 最少机器数调度所有任务
- 贪心：按 start 排序，用最小堆维护各机器的 end

### 2.5 Huffman 编码 ⭐
https://www.bilibili.com/video/BV1qu411F7Zs/?spm_id_from=333.337.search-card.all.click&vd_source=0a34946acd4b87aa5c928be2fa1376a1

**问题**：给定字符频率，构造最短前缀编码

**算法**：
1. 每个字符建叶子节点，频率为权重
2. 每次从最小堆取两个最小权重节点
3. 合并为新节点（权重 = 两者之和），放回堆
4. 不断的合并最小的两个，重复直到只剩一棵树
![[截屏2026-04-30 23.36.54.png]]
出现次数最多的码，越短（越靠上）
- **复杂度**：O(n log n)
- **正确性**：频率最低的两个字符必在最深层 → 交换论证

### 2.6 Dijkstra / Prim / Kruskal

都是贪心算法（见 Module 12、13）。

## 3. 贪心 vs DP vs 回溯

| 问题 | 方法 | 原因 |
|---|---|---|
| 分数背包 | **贪心** | 可拆分，价值密度排序即可 |
| 0-1 背包 | **DP** | 离散决策，贪心反例多 |
| 硬币找零（美元面额） | **贪心** | 面额系统"规范" |
| 硬币找零（任意面额） | **DP** | 贪心可能失败（如 1,3,4 找 6） |
| 活动选择 | **贪心** | 按 end 排序 |
| 最长递增子序列 | **DP** | 贪心仅能得到长度不是序列 |

---

## 4. 常考陷阱

- ❌ "贪心"不等于"简单" —— 证明正确性往往最难
- ❌ 0-1 背包按价值密度贪心 **错误**
- ❌ Huffman 要求频率非负，频率为 0 的字符不参与
- ❌ 活动选择按 **end** 排序，不是 start 或 duration

## 5. 面试高频题

1. **跳跃游戏 II** —— 每步跳到能到达的最远处
2. **加油站** —— 总油量 ≥ 总消耗时贪心找起点
3. **分发糖果** —— 两次扫描
4. **会议室 II** —— 按 end 排序 + 最小堆
5. **Huffman 编码手画** —— 给定频率构造树 + 写出编码
6. **区间合并** —— 按 start 排序后合并

## 6. 三视角职业分析

- **工程师视角**：大多数场景的"第一感觉"解法，但务必证明或给反例
- **研究员视角**：Matroid 理论统一解释"哪些问题贪心可解"
- **系统架构师视角**：任务调度器（k8s 调度、数据库查询计划器）充满贪心启发
