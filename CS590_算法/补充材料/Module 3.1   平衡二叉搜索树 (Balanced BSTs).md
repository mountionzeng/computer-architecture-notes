## 1. 思考脉络

1. BST 最坏 O(n) → 想一种方法自动保持"矮胖" → **平衡**
2. "每次插入/删除后检测失衡并修正" → 核心工具：**旋转（Rotation）**
3. 不同"平衡定义" → 不同树家族：
   - 严格高度平衡 → **AVL**
   - 颜色约束（更宽松）→ **Red-Black**
   - 访问局部性 → **Splay**![[截屏2026-04-28 18.52.23.png]]

**BST 性质**是普通 BST 就有的，只要插入和删除时按规则操作就自动保持。**平衡 BST 真正解决的问题是第二层——保证高度始终 O(log n)**。

旋转这个工具的意义是：**它能调整树的形状（让高度变矮），同时不破坏 BST 性质**。所以旋转不是用来"达到 BST 性质"的，而是用来"在保持 BST 的前提下，让树变平衡"的。

明确了这个，我们看代码层面怎么做。我用 AVL 树来讲，因为它的逻辑最直接最清晰；红黑树和 Splay 是同一思路的变体。

### 2.1 旋转（Rotation）

### 2.2 AVL 树

- **平衡因子**：`BF(v) = height(left) - height(right)` ∈ {-1, 0, 1}
- 插入 → 回溯检查 BF → 如果越界用四种旋转修复：LL / RR / LR / RL
- 高度严格 ≤ 1.44 log(n+2)
- **优点**：查找常数更快；**缺点**：插入旋转次数多

**关键理解**：AVL 插入分两个阶段——
1. **下降**：和普通 BST 完全一样，递归找到插入位置
2. **回溯**：函数调用栈 unwind 时，每层都更新高度、检查平衡因子；一旦 `|BF| > 1` 就地旋转，**旋转完高度恢复，上面的祖先不再受影响**，所以 AVL 插入最多只需 1 次旋转

### 2.3 Red-Black 树（红黑树）
![[Pasted image 20260428192023.png]]

五条性质：
1. 每节点红或黑
2. 根是黑
3. 叶（NIL）是黑
4. 红节点的孩子是黑（不能红红相邻）
5. 从任一节点到其所有后代叶，**黑节点数相同**

- 高度 ≤ 2 log(n+1)
- 插入/删除最多 **O(1) 次旋转**（AVL 可能 O(log n)）
- 是 `std::map` / Java `TreeMap` / Linux CFS 调度器的底层

### 2.4 Weak AVL（rank-balanced）

- 每节点有 rank 值
- 更宽松的约束，统一 AVL 和 Red-Black 的证明框架

### 2.5 Splay 树

- 每次访问节点 x，通过一连串旋转把 x **旋到根**
- **Zig / Zig-Zig / Zig-Zag** 三种操作
- 单次最坏 O(n)，但**摊还 O(log n)**
- 自适应：热数据自动靠近根

## 3. 平衡树对比表

| 特性 | AVL | Red-Black | Splay |
|---|---|---|---|
| 查找 | **最快** | 稍慢 | 摊还 O(log n) |
| 插入 | 多次旋转 | **≤2 次旋转** | 摊还 O(log n) |
| 平衡程度 | 严格 | 较宽松 | 自适应 |
| 空间开销 | 1 int | 1 bit | 无额外（但改指针频繁）|
| 典型应用 | 内存数据库 | STL map、内核 | 缓存、LRU |

---

## 4. 常考陷阱

- ❌ AVL 删除可能需要**一路旋转到根**（O(log n) 次旋转）
- ❌ Red-Black 插入新节点默认染**红**（避免违反黑高）
- ❌ 旋转保持 BST 性质只要"中序序列不变"
- ❌ Splay 的"单次最坏 O(n)"不是算法缺陷，是摊还分析的对象
